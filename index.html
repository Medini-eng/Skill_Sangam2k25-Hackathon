24 Hrs Overnight Hackathon | 2nd & 3rd June 2025 | Sahyadri Engineering College

Welcome to Skill Sangam 2K25

Are you ready to challenge yourself? Explore the event details and get ready for the big reveal!

Event Flow & Agenda

1. Registration & Welcome

  • Time: 8:30 AM - 9:30 AM
  • Location: Registration Booth near Seminar Hall Ground Floor
  • Task: Complete the registration process and collect the Welcome Kit.
  • Note: Limited on-spot registration will be allowed.

2. Inauguration Ceremony

  • Time: 9:30 AM - 10:30 AM
  • Location: Seminar Hall Ground Floor
  • Activity: Opening speech by Chief Guests. Briefing about the Hackathon rules and judging process.

3. Team Allocation & Kickoff

  • Time: 10:30 AM
  • Task: Teams move to allotted rooms as per the registration details.
  • Objective: Settle in the workspace. Connect with team mentors. Finalize problem statement.

4. Hackathon Launch

  • Time: 11:00 AM
  • Event Launch: Begin work on the selected problem statement.

Instructions & Milestone Checkpoints

  • The available components from the Skill Labs can be used for the hackathon.
  • Each problem statement can be taken by a maximum of 5 teams. Extra components must be brought by students for the event.

Checkpoint 1 – Ideation & Problem Presentation

  • Time: 12:30 PM
  • Task: Submit your initial idea and problem understanding. Short presentation to mentors on proposed solutions and technologies.
  • Focus Areas: Clarity of the problem, innovation in solution, component & technology plan.

Checkpoint 2 – Technical Progress Review

  • Time: 5:30 PM
  • Task: Demonstrate hardware & circuit progress. Explain selected electronic components. Share team coordination, work distribution, and features implemented. Bonus for innovative elements added to the base idea.
  • Focus Areas: Circuit building & microcontroller programming, UI/UX or dashboard components, early demo of prototype if possible.

Midnight Mentoring & Evaluation

  • Time: Post 12:00 AM
  • Task: Get feedback from industry mentors. Adjust technical aspects based on expert inputs.
  • Tip: Use this session to elevate the project to an MVP (Minimum Viable Product).

Late Night Fun & Engagement

  • Games, icebreakers, and surprise snacks to boost team spirit!

Day 2 - Final Phase

  • Checkpoint 3 – Final Selection
  • Time: 9:30 AM (Next Day)
  • Task: Present complete working prototype (MVP). Final technical demonstration and documentation submission.

Judging Criteria

  • Real-time functionality
  • Component utilization
  • Coding and integration
  • Team dynamics
  • Product impact

Real-Time Voice Transfer and Human Speech Filter

Develop a real-time voice transfer system using two ESP32 boards. Connect two microphones to the first ESP32 and a speaker to the second ESP32 board. Implement a real-time voice transfer system between the two ESP32 boards. Utilize Bluetooth communication protocols for seamless data transfer. Minimize latency to achieve a smooth and real-time voice transfer experience. Design and Implement a Filter to selectively transmit only Human speech to the receiving ESP32/STM.

Smart Inventory & Shelf Management System

Build a system that tracks stock levels on shelves using weight sensors or RFID. Send alerts for restocking and generate usage analytics on a cloud dashboard.

Remote Biometric Enrollment & Smart Machine Access System

This system uses an ESP32 with an R307 fingerprint sensor to control machine or door access based on biometric authentication. A Raspberry Pi acts as the central server for remote fingerprint enrollment, access monitoring, and shift log tracking via a web dashboard. Real-time data is synced over Wi-Fi using HTTP/MQTT protocols. This setup ensures secure access control and efficient employee shift monitoring.

Industry 4.0 – In-Vehicle Subsystem Communication for Smart Dashboards

Develop an STM-based embedded system to simulate communication between multiple in-vehicle subsystems (e.g., speed sensor, fuel monitor, infotainment, temperature control) and a central smart dashboard. The system should demonstrate real-time data exchange, fault tolerance, and subsystem isolation using a communication protocol like CAN or UART.
